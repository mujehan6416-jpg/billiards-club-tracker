# 당구 동호회 경기 기록 & 대시보드 (PWA) — 설계 문서

> 작성일: 2026-06-14
> 최종 수정일: 2026-06-14

## 1. 개요

당구 동호회 모임에서 진행한 **3구(스리쿠션) 1:1 경기 결과**를 기록하고,
회원 전적·상대전적·개인 추이를 **대시보드**로 보며, 현장에서 **1:1 자동매칭**을
돌릴 수 있는 개인용 앱.

- **운영 모델**: 단일 운영자(본인)가 현장에서 모든 데이터를 입력. 로그인·계정·권한 없음.
- **형태**: 서버 없는 순수 클라이언트 **PWA** (홈화면 설치, 오프라인 동작).
- **공유**: 일방향 내보내기(대진표/결과 이미지·텍스트, 데이터 백업 파일).

## 2. 기술 스택 & 구조

| 항목 | 선택 | 이유 |
|------|------|------|
| 형태 | PWA (vite-plugin-pwa) | 앱스토어 불필요, iOS/안드 공통, 오프라인 |
| 프레임워크 | React + Vite | 가볍고 PWA 설정 자동, 개인 프로젝트 적합 |
| 저장소 | localStorage (단일 JSON 상태트리) | 데이터량 작음, 백업이 곧 상태 통째 export |
| 백업 | JSON/CSV 내보내기 → 폰 공유시트로 Drive/Sheets 저장 | OAuth 없이 안전, CSV는 시트서 바로 열림 |
| 공유 | 대진표/결과 → 이미지(canvas)·텍스트 export | 카톡 등 현장 공유 |

구글 시트 API 직접 자동동기화는 OAuth 부담으로 **2단계(추후)**. MVP는 파일 내보내기.

화면(하단 탭 4개): `회원 │ 모임(오늘) │ 대시보드 │ 설정`

**모임 탭 UI: 코트 그리드형** (확정). 당구대를 카드 그리드(예: 2열)로 배치해 각
테이블의 현재 대전을 보여줌. 빈 테이블 카드는 강조 표시 + "매칭 추천" 버튼 →
추천된 짝을 배정. 카드 탭하면 득점 입력. 하단에 대기자 목록 표시. 5테이블 동시
진행과 "빈 자리 채우기" 흐름에 직관적으로 맞음.

## 3. 경기 규칙 (도메인 핵심)

종목: **3구 1:1**. 각 선수는 **핸디 = 목표 개수**를 가짐 (예: A=25, B=20). 경기는 1시간 단위.

**달성률 = 친 개수 ÷ 핸디** 로 승패 판정:

1. **시간 내 핸디 달성** → 먼저 달성한 사람 즉시 승, 경기 종료 (달성률 100%).
2. **시간 종료(미달성)** → 달성률 높은 사람 승.
   - 예: A 20/25 = 80%, B 15/20 = 75% → A 승.
3. **달성률 동일** → 무승부.

> 핵심: "누가 더 많이 쳤나"가 아니라 "핸디 대비 몇 % 쳤나"로 판정 → 핸디 달라도 공평.

**시간 강제 금지**: 앱은 1시간 자동 종료를 강제하지 않음. 경기는 운영자가 결과(득점)를
입력할 때 끝남. 타이머는 참고용 스톱워치(선택)일 뿐 강제 종료 안 함. (늦게 합류한 경기도
다른 경기 타이밍에 맞춰 끝나므로 일괄 컷오프하면 안 됨.)

## 4. 데이터 모델

```
AppState
├─ members[]
│   ├─ id, name
│   ├─ handicap            현재 핸디(=목표 개수)
│   ├─ handicapHistory[]   { value, changedAt } 변경 이력
│   └─ active              탈퇴/휴면 (삭제 대신 false → 과거 전적 보존)
│
├─ sessions[]
│   ├─ id, date
│   ├─ attendeeIds[]       그날 참석자
│   └─ games[]
│        ├─ id
│        ├─ playerAId, playerBId
│        ├─ handicapA, handicapB   그 경기 시점 목표(스냅샷, 회원 핸디서 자동 채움·수정가능)
│        ├─ scoreA, scoreB         실제 친 개수
│        ├─ endType                'cleared'(핸디달성) | 'time'(시간종료)
│        └─ playedAt               매칭/기록 순서·시각
│
└─ settings                마지막 백업 시각 등
```

- **승패는 저장하지 않고 점수+핸디로 계산** (`rate = score / handicap`, 높은 쪽 승, 같으면 무승부) → 규칙 바뀌어도 재계산 가능.
- **핸디를 경기마다 스냅샷 저장**: 회원 핸디는 시간이 지나며 바뀌므로(15→20) 과거 경기를 당시 핸디로 정확히 재현하려면 필요. 입력 시 회원 현재 핸디 자동 채움, 수정 가능.

## 5. 자동매칭 로직

**모델: 풀(pool) 기반 유연 매칭** (라운드 고정 X, 이벤트 기반).

- 참석자 상태: `대기 / 경기중` 토글.
- 두 가지 진입점:
  - **대기자 전체 매칭**: 시작 시 빈 테이블 전부를 한 번에 배정(그리디).
  - **다음 대전 추천**: 한 경기 끝나면 그 자리만 재매칭.
- 늦게 합류한 인원은 대기 풀에 추가 → 다른 경기 끝나는 타이밍에 자연 합류.

**매칭 우선순위:**
1. 누적 맞대결 횟수(`meetCount[a][b]`) 적은 순 — *가장 안 만난 짝 우선*.
2. 오늘 경기 적게 한 순 — 공평(많이 쉰 사람 먼저).
3. 랜덤 타이브레이크.

- 홀수 인원이면 1명 자동 휴식(적게 쉰 사람 우선 매칭되어 자연 순환).
- 추천은 **제안일 뿐 강제 아님** — 운영자가 수동 지정/교체 가능.
- **핸디 차이는 매칭에 미반영** (달성률로 공평해지므로).
- 한 모임 = 보통 2경기/인 (강제 X, 대시보드에 "오늘 N경기" 표시로 확인).

## 6. 대시보드

| 종류 | 내용 |
|------|------|
| 승률 랭킹 | 회원별 경기·승·패·무·승률·**평균 달성률** 정렬표 |
| 날짜별 기록 | 모임 선택 → 참석자·경기·결과 타임라인 |
| 상대전적 | 두 회원 선택 → 누적 맞대결 + 달성률 비교 |
| 개인 추이/연승 | 회원 선택 → 시간순 달성률·승률 그래프, 연승 기록 |

**핸디 적정성 힌트**: 평균 달성률이 꾸준히 100% 근처면 "핸디 올릴 때", 낮으면 "내릴 때"
→ 수동 핸디 조정 판단 근거로 활용.

## 7. 백업 & 공유

- **백업 내보내기**: `AppState` 통째 → JSON + CSV(구글 시트 호환). 폰 공유시트로 Drive/Sheets 저장.
- **가져오기**: JSON 선택 → 복원 (폰 교체·캐시삭제 대비).
- **현장 공유**: 오늘 대진표/결과 → 이미지(canvas) 또는 텍스트 export → 카톡 공유.

## 8. 범위 (YAGNI)

**포함(MVP):** 회원·핸디 관리, 모임/참석, 경기 득점 기록, 달성률 승패, 풀 기반 자동매칭,
4종 대시보드, JSON/CSV 백업, 이미지/텍스트 공유.

**제외(추후):** 구글 시트 API 자동동기화, 멀티유저/로그인, 4구·포켓볼 등 타 종목,
서버·클라우드, 강제 타이머.
